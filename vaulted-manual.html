Operator Guide

A Mac front end for encrypted recovery files that should never live in plaintext sync folders.

Vaulted keeps the encrypted archive on `dees-workbench`, opens it from a Mac into a temporary RAM disk, lets Finder handle ordinary edits, then saves the RAM disk back into a new encrypted archive and ejects it. The project code lives in `/srv/cloud-mirror/vaulted`; encrypted vault files live outside the project under `/srv/cloud-mirror/secure-vaults`.

Golden Rule

Do not store recovery codes, passphrases, `.gpg` vaults, or decrypted vault contents inside the Vaulted repo, Google Drive tooling folders, or ordinary cloud-mirror project folders.

Vault Architecture

Feature

Choose Vault

Selects a `.gpg` archive from the secure-vaults mirror and maps Mac paths to the matching `/srv/cloud-mirror/secure-vaults` server path.

Feature

Secure Prompt

The passphrase is entered in the app UI and sent to the helper over stdin. It is not placed on the process command line.

Feature

RAM Editing

Finder opens a temporary RAM disk for ordinary file edits. Cloud sync folders do not receive plaintext vault contents.

Feature

Auto-Lock

Lock requests run on button press, app quit, window close, system sleep, or the default 300-second auto-lock timer.

Operations

Common Commands

Check Mac and Helper Activity

ps -axo pid,ppid,stat,command | rg '/Volumes/cloud-mirror/vaulted|Vaulted\\.app|open-recovery-vault-ramdisk\\.sh|/Volumes/RecoveryVault' || true
hdiutil info | rg '/Volumes/RecoveryVault|RecoveryVault' || true
mount | rg 'RecoveryVault' || true

Check Remote Locks

ssh -n -T -o LogLevel=ERROR dee@dees-workbench.local \
  "find /srv/cloud-mirror/secure-vaults/.locks -maxdepth 2 -print 2>/dev/null | sort"

Build Native App

swiftc /Volumes/cloud-mirror/vaulted/vaulted-mac-app/Sources/main.swift \
  -o /Volumes/cloud-mirror/vaulted/Vaulted.app/Contents/MacOS/Vaulted \
  -framework AppKit

Package Safely

find /Volumes/cloud-mirror/vaulted -name '._*' -type f -delete
find /Volumes/cloud-mirror/vaulted -name '.DS_Store' -type f -delete
codesign --force --deep --sign - /Volumes/cloud-mirror/vaulted/Vaulted.app
cd /Volumes/cloud-mirror/vaulted
COPYFILE_DISABLE=1 zip -qry -X Vaulted.app.zip Vaulted.app
Recovery

Failure Handling

01

If encryption or verification fails, do not eject or delete the RAM disk. Keep it mounted and fix the save path first.

02

If the Mac loses power while open, unsaved RAM-disk edits may be lost. The previous encrypted archive should still be intact.

03

If a stale lock blocks opening, confirm no Mac has the vault open before removing the lock directory.

04

If Finder creates AppleDouble sidecars, clean `._*` and `.DS_Store` before committing or packaging.
